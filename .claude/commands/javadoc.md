Add meaningful Javadoc to public and protected API surfaces in recently changed Java files.

**Steps:**
1. Run `git diff --name-only HEAD` to find changed Java files. If nothing is staged/committed yet, run `git diff --name-only` to find unstaged changes. Work only on `.java` files.
2. For each file, read it and identify which elements need Javadoc:
   - **Always document**: public/protected classes, interfaces, enums, records, and their public/protected methods and constructors — if they don't already have a Javadoc comment
   - **Skip `@Override` methods** — unless the override meaningfully changes the contract (e.g. narrows the type, adds a side effect, or has a non-obvious constraint worth calling out)
   - **Skip private and package-private members** — they're implementation details
   - **Skip getters/setters** generated by Lombok (`@Getter`, `@Setter`) — the annotation is self-documenting
   - **Skip trivial constructors** that do nothing beyond storing injected dependencies
3. Write Javadoc that explains **intent and contract**, not the implementation:
   - Lead with a one-sentence summary that says *what* the element does from the caller's perspective
   - Add `@param` for each parameter that isn't self-evident from its name and type
   - Add `@return` when the return value has non-obvious semantics (skip for `void` and trivial getters)
   - Add `@throws` for checked exceptions or for unchecked exceptions that callers are expected to handle
   - Avoid restating the method name in prose ("Returns the id" for `getId()` adds nothing)
   - Use present-tense, third-person singular: "Creates a…", "Returns the…", "Validates that…"
4. Edit each file in place with the new/updated Javadoc.
5. After all files are updated, run `./gradlew compileJava compileTestJava` to confirm no compilation errors.

**Tone and style:**
- Concise over verbose — one sentence is often enough
- No `@author`, `@since`, `@version` tags
- Do not add `{@inheritDoc}` to `@Override` methods — if documenting an override is warranted, write the specific contract directly
